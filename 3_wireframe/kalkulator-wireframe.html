Anotasi

② Main display menampilkan "0" sebagai nilai awal

Screen 2 — State Input_A INPUT ANGKA 1
┌─────────────────────────┐ │ KALKULATOR SIMPLE │ ├─────────────────────────┤ │ [ ekspresi kosong ] │ ← sub display │ │ │ [ 84 ] │ ← menampilkan "84" ├─────────────────────────┤ │ [ C ] [ CE ] [ ÷ ] │ │ [ 7 ] [ 8* ] [ 9 ] │ ← * = baru ditekan │ [ 4* ] [ 5 ] [ 6 ] │ ← * = baru ditekan │ [ 1 ] [ 2 ] [ 3 ] │ │ [ 0 ] [ . ] │ │ [ = ] │ ├─────────────────────────┤ │ [ + ] [ - ] [ × ] │ └─────────────────────────┘
Anotasi
① User menekan 8 lalu 4 → display menampilkan "84"
② Sub display masih kosong — operator belum dipilih
③ Setiap digit baru di-append ke kanan display
④ CE aktif → tekan CE hapus digit terakhir (84 → 8)
⑤ State: INPUT_A, siap menerima operator
Screen 3 — State Operator_Set OPERATOR DIPILIH
┌─────────────────────────┐ │ KALKULATOR SIMPLE │ ├─────────────────────────┤ │ [ 84 + ] │ ← sub display aktif │ │ │ [ 84 ] │ ← main display tetap ├─────────────────────────┤ │ [ C ] [ CE ] [ ÷ ] │ │ [ 7 ] [ 8 ] [ 9 ] │ │ [ 4 ] [ 5 ] [ 6 ] │ │ [ 1 ] [ 2 ] [ 3 ] │ │ [ 0 ] [ . ] │ │ [ = ] │ ├─────────────────────────┤ │ [+*] [ - ] [ × ] │ ← + aktif/terpilih └─────────────────────────┘
Anotasi
① Sub display menampilkan "84 +" — ekspresi sedang berjalan
② Main display tetap "84" hingga angka kedua ditekan
③ Tombol + ditandai aktif/terpilih (bisa pakai highlight)
④ Tekan operator lain → ganti operator, tidak kalkulasi
⑤ State: OPERATOR_SET, menunggu angka kedua
Screen 4 — State Input_B INPUT ANGKA 2
┌─────────────────────────┐ │ KALKULATOR SIMPLE │ ├─────────────────────────┤ │ [ 84 + ] │ ← sub display │ │ │ [ 36 ] │ ← angka kedua ├─────────────────────────┤ │ [ C ] [ CE ] [ ÷ ] │ │ [ 7 ] [ 8 ] [ 9 ] │ │ [ 4 ] [ 5 ] [ 6 ] │ │ [ 1 ] [ 2 ] [ 3 ] │ │ [ 0 ] [ . ] │ │ [=* ] │ ← = siap ditekan ├─────────────────────────┤ │ [ + ] [ - ] [ × ] │ └─────────────────────────┘
Anotasi
① Main display berganti ke "36" (angka kedua)
② Sub display tetap "84 +" sebagai konteks
③ Tombol = aktif dan siap dieksekusi
④ CE aktif → hapus digit terakhir angka kedua
⑤ State: INPUT_B, siap kalkulasi
Screen 5 — State Result HASIL DITAMPILKAN
┌─────────────────────────┐ │ KALKULATOR SIMPLE │ ├─────────────────────────┤ │ [ 84 + 36 = ] │ ← ekspresi lengkap │ │ │ [ 120 ] │ ← hasil kalkulasi ├─────────────────────────┤ │ [ C ] [ CE ] [ ÷ ] │ │ [ 7 ] [ 8 ] [ 9 ] │ │ [ 4 ] [ 5 ] [ 6 ] │ │ [ 1 ] [ 2 ] [ 3 ] │ │ [ 0 ] [ . ] │ │ [ = ] │ ├─────────────────────────┤ │ [ + ] [ - ] [ × ] │ └─────────────────────────┘
Anotasi
① Sub display: "84 + 36 =" — riwayat ekspresi lengkap
② Main display: "120" — hasil akhir
③ Tekan operator → 120 jadi operand1 (kalkulasi berantai)
④ Tekan angka → mulai kalkulasi baru dari awal
⑤ Tekan = lagi → ulangi operasi (120 + 36 = 156)

Anotasi
① Sub display: "84 ÷ 0 =" — menampilkan ekspresi penyebab error
② Main display: "Error!" — teks merah / warna peringatan
③ Semua tombol dinonaktifkan kecuali C
④ Tombol "---" = disabled / tidak bisa ditekan
⑤ Satu-satunya jalan keluar: tekan C → kembali ke IDLE
Detail Komponen
Komponen: Display 2 LAYER
┌──────────────────────────────┐ │ AREA DISPLAY │ │ │ │ ┌────────────────────────┐ │ │ │ [ sub display ] │ │ ← 14px, right-align │ │ ekspresi / riwayat │ │ warna muted/abu │ └────────────────────────┘ │ │ │ │ ┌────────────────────────┐ │ │ │ │ │ │ │ 0 │ │ ← 32px bold, right-align │ │ main display │ │ warna dominan │ └────────────────────────┘ │ │ │ └──────────────────────────────┘ Overflow handling: Jika > 9 digit → font mengecil otomatis Jika > 15 digit → notasi: 1.23456789e+12
Spesifikasi
① Sub display: font kecil, rata kanan, warna sekunder
② Main display: font besar bold, rata kanan, warna primer
③ Keduanya tidak bisa di-edit langsung oleh user (read-only)
④ Overflow: font mengecil jika angka terlalu panjang
Komponen: Grid Tombol 4×4 + KHUSUS
Grid tombol: 3 kolom utama ┌─────┬─────┬─────┐ │ C │ CE │ ÷ │ ← baris kontrol ├─────┼─────┼─────┤ │ 7 │ 8 │ 9 │ ← baris 1 ├─────┼─────┼─────┤ │ 4 │ 5 │ 6 │ ← baris 2 ├─────┼─────┼─────┤ │ 1 │ 2 │ 3 │ ← baris 3 ├─────┴─────┼─────┤ │ 0 │ . │ ← 0 = lebar 2 kolom └───────────┴─────┘ │ = │ ← di luar grid utama └─────────────────┘ ┌─────┬─────┬─────┐ │ + │ - │ × │ ← baris operator bawah └─────┴─────┴─────┘ Tipe tombol: [CTRL] = C, CE → warna berbeda [OPR] = + - × ÷ → warna berbeda [NUM] = 0-9, . → warna netral [EQL] = = → warna aksen/utama
Spesifikasi
① 4 tipe tombol dengan warna berbeda sebagai penanda fungsi
② Tombol 0 selebar 2 kolom — lebih mudah ditekan
③ Tombol = menonjol dengan warna aksen / berbeda
④ Semua tombol memiliki efek hover dan active (press)
